JS Frontend Frameworks

    Why JS Frontend Frameworks

    • Libraries or tools that provide developers with the necessary structure and utilities to build interactive user interfaces for web applications.
    • These frameworks help streamline the development process by offering features like data binding, routing, state management, and component-based architectures.

    Popular JS Frontend Frameworks

    • React:

      • Developed by Facebook, React is a declarative, efficient, and flexible JavaScript library for building user interfaces.
      • It uses a component-based architecture and a virtual DOM for efficient rendering.

    • Angular:

      • Developed and maintained by Google, Angular is a comprehensive framework for building web applications.
      • It provides a complete solution with features like two-way data binding, dependency injection, and routing.

    • Vue.js:

      • Progressive JavaScript framework for building user interfaces. It's known for its simplicity and flexibility, allowing developers to incrementally adopt its features into existing projects.
      • Vue.js is often praised for its small size and ease of learning.

    • Svelte:

      • Framework that shifts much of the work to compile time, resulting in smaller bundle sizes and potentially faster runtime performance.
      • It allows developers to write components with a syntax similar to HTML, CSS, and JavaScript, and then compiles them to highly efficient imperative code.

    React

    • An open-source JavaScript library developed by Facebook for building user interfaces, particularly for single-page applications.
    • It's often used for building interactive and dynamic user interfaces for web and mobile applications.
    • React is known for its component-based architecture, where the user interface is broken down into reusable components.
    • Each component manages its own state and can be composed together to create complex UIs.
    • This approach makes it easier to maintain and scale large applications since developers can focus on building and managing smaller, self-contained components.

    Key features of React

    • Virtual DOM:

      • React uses a virtual DOM to efficiently update the user interface.
      • Instead of directly manipulating the DOM, React creates a virtual representation of it in memory and updates only the parts of the actual DOM that have changed, minimizing browser reflows and improving performance.

    • JSX:

      • JSX is a syntax extension that allows developers to write HTML-like code within JavaScript.
      • This makes it easier to create React components by blending HTML markup with JavaScript logic.

    • Component-based architecture:

      • React encourages a modular approach to building user interfaces, where UIs are broken down into reusable components.
      • This promotes code reusability and maintainability.

    • Unidirectional data flow:

      • React follows a unidirectional data flow model where data flows in a single direction, from parent to child components.
      • This helps in maintaining predictable state changes and simplifies debugging.

    • React Hooks:

      • Hooks are functions that allow developers to use state and other React features without writing classes.
      • They provide a more functional approach to managing component state and side-effects.

    • React Router:

      • Popular routing library for React applications.
      • It allows developers to define multiple routes in their applications and handle navigation between different views.

    ES6

    • ECMAScript 6 (ES6), also known as ECMAScript 2015, brought significant improvements to JavaScript, which have had a profound impact on the development of React applications.

    Classes

    • ES6 classes provide a more intuitive and cleaner syntax for defining objects and handling inheritance, which is especially beneficial in the context of React components.

    Arrow Functions

    • Arrow functions provide a shorter syntax for writing functions.

    Map function

    • The .map() method is a powerful array method in JavaScript that is commonly used in React to transform arrays of data into arrays of React elements.
    • This method is particularly useful when rendering lists of components.

    Destructuring

    • Destructuring allows for easier extraction of values from arrays or properties from objects, which is particularly handy when working with React props and state.

    Spread operator

    • The spread operator (...) is a powerful feature in JavaScript, introduced in ES6, that allows for the expansion of iterable elements (such as arrays and strings) and properties of objects into individual elements.

    Module

    • ES6 modules allow for a modular approach to organizing code, making it easier to manage and reuse components in React.

    JSX

    • JSX, or JavaScript XML, is a syntax extension for JavaScript commonly used with React to describe what the UI should look like.
    • JSX may resemble HTML, but it has full power of JavaScript.
    • It is compiled down to JavaScript by tools like Babel before being executed in the browser.

    Components

    • React components are the fundamental building blocks of a React application.
    • They encapsulate pieces of the user interface and manage their own state, making it easier to build complex UIs by breaking them down into smaller, reusable parts.

    Function components

    • Function components are the simplest way to write components in React.
    • They are JavaScript functions that accept props as an argument and return JSX.

    Class components

    • Class components are more feature-rich and allow you to use additional React features such as local state and lifecycle methods.

    Function components are preferred over Class components

    • Function components are simpler and more concise than class components. They eliminate the need for boilerplate code such as the constructor and the render method.
    • Hooks, introduced in React 16.8, allow you to use state and other React features in function components. Hooks like useState, useEffect, useContext, and others provide powerful capabilities that were previously only available in class components.
    • Function components can have a smaller memory footprint compared to class components because they don’t require an instance of a component to be created.
    • Function components don’t use the this keyword, which avoids common pitfalls related to its usage and binding. This makes the code easier to understand and debug.
    • Function components are easier to test because they are plain JavaScript functions. They don’t require complex setup and teardown that might be needed for class components.
    • Hooks promote better composition and reuse of stateful logic. Custom hooks allow you to extract component logic into reusable functions.

    Props

    • Are a fundamental concept in React that allow you to pass data from a parent component to a child component.
    • Props are read-only and help make components reusable by allowing them to receive dynamic data.

    Prop Drilling

    • Prop drilling is a common issue in React applications where data is passed from a parent component to a deeply nested child component through intermediate components that do not need the data themselves.
    • This can lead to components becoming cluttered with props that are only passed down to children, making the code harder to maintain and understand.

    Events

    • Handling events in React is similar to handling events in plain JavaScript, but with some syntactic and functional differences.
    • React events are named using camelCase, and you pass event handler functions as props to elements.

    Event Types

    • Clipboard events: onCopy onCut onPaste
    • Form events: onChange onInput onSubmit
    • Keyboard events: onKeyDown onKeyPress onKeyUp
    • Mouse events: onClick onDoubleClick onMouseDown onMouseEnter onMouseLeave onMouseMove onMouseOut onMouseOver onMouseUp
    • Touch events: onTouchCancel onTouchEnd onTouchMove onTouchStart
    • Focus events: onFocus onBlur

    Conditionals

    • The logical && operator can be used to conditionally include an element or component. If the condition is true, the element after && will be rendered.
    • The ternary operator can be used to conditionally render one of two elements.
    • An if statement and checking the value of a variables can be used to render elements.

    Lists

    • The map() method allows you to iterate over an array and return a new array of React elements.
    • Each child in a list should have a unique "key" prop to help React identify which items have changed, added, or removed. This improves performance by aiding React in efficient DOM updates.
    • Do not use array indices as the keys.

    Forms

    • In React, form elements like input, textarea, and select typically maintain their own state based on user input. However, you can also control the state of these form elements by making them controlled components. Controlled components allow React to have full control over the form elements' state.
    • Reack hook useRef can be used to get a reference to a DOM element, which can be useful for managing focus, text selection, or triggering imperative animations.

    Hooks

    • React Hooks are functions that let you use state and other React features without writing a class.
    • They were introduced in React 16.8 to enable functional components to have state and side effects, which previously required class components.
    • Hooks provide a more direct API to the React concepts you already know (state, lifecycle, context, refs, etc.).

    useState

    • Allows you to add state to functional components.
    • It's one of the most commonly used hooks because it provides a simple way to manage and update state in your components.

    useContext

    • Allows you to subscribe to React context without introducing nesting.
    • Context provides a way to pass data through the component tree without having to pass props down manually at every level.
    • It is typically used when some data needs to be accessible by many components at different nesting levels, such as theme, user information, or preferred language.
    • It serves the same purpose as lifecycle methods like componentDidMount, componentDidUpdate, and componentWillUnmount in class components.

    useEffect

    • Allows you to perform side effects in function components. Side effects can include things like data fetching, manually changing the DOM, and setting up subscriptions.
    • The useEffect hook is called inside a component to perform these side effects after the component renders.

    useRef

    • Returns a mutable ref object whose .current property is initialized to the passed argument (initialValue).
    • This object persists for the full lifetime of the component.
    • Refs are commonly used to access a DOM element directly or to persist a mutable value without causing a re-render.

    useMemo

    • Memoizes the result of a calculation, allowing you to optimize the performance of your components by skipping expensive calculations on every render if the inputs haven't changed.
    • It returns a memoized value that only recomputes when one of its dependencies has changed.

    useReducer

    • Allows you to manage complex state logic in functional components with a more predictable and controlled way compared to useState.
    • It's inspired by the reducer pattern used in Redux.
    • While useState is simpler and more suitable for managing local component state, useReducer becomes useful when the state logic becomes more complex or when the next state depends on the previous one.

    Custom Hook

    • Allow you to extract and reuse logic between different components.
    • They are simply JavaScript functions that use other hooks and can encapsulate stateful logic.
    • Custom hooks help to keep your code clean and modular by abstracting complex logic out of your components.

    React Router

    • Popular library for managing routing in React applications.
    • It enables developers to create single-page applications with multiple views, allowing navigation between different components or pages while maintaining a consistent user experience.
    • React Router provides a declarative / programmatic approach to routing, allowing developers to define routes as components and easily handle navigation and URL changes.

    Core Components

    • BrowserRouter: A component that uses the HTML5 history API to keep the UI in sync with the URL.
    • Route: A component that renders some UI when its path matches the current URL.
    • Link: A component to navigate to a different route defined in the application.

    Rendering Strategies

    • CSR, SSR, SSG, and ISR are different approaches to rendering web pages, each with its own advantages and use cases.

    Client-Side Rendering (CSR) - React

    • In CSR, the initial HTML is minimal, and JavaScript is responsible for rendering the content on the client side.
    • When a user visits a page, the browser downloads the HTML file, and then JavaScript is executed to fetch data from an API and render the content dynamically.
    • Pros: Fast initial load time, dynamic content updates without full page reloads, better user interactivity.
    • Cons: SEO challenges, slower time to first meaningful paint (TTTFP), reliance on JavaScript for rendering.

    Server-Side Rendering (SSR) - React + Next.js or Gatsby.js

    • In SSR, the server generates the complete HTML for a page and sends it to the client, where it's directly rendered by the browser.
    • When a user visits a page, the server fetches the necessary data, renders the page, and sends the fully-rendered HTML to the client.
    • Pros: Improved SEO, faster time to first paint (TTFP), better performance on slow client devices or networks.
    • Cons: Higher server load, potential for longer response times due to server-side rendering overhead.

    Static Site Generation (SSG) - React + Next.js or Gatsby.js

    • In SSG, pages are pre-built at build time, and the resulting HTML files are served to clients without the need for server-side rendering on each request.
    • During the build process, the server fetches data, generates HTML pages for each route, and stores them as static files.
    • Pros: Blazing-fast load times, reduced server load, improved security, better SEO.
    • Cons: Content may become stale until the next build, dynamic content updates require rebuilding, limited dynamic capabilities.

    Incremental Static Regeneration (ISR) - React + Next.js

    • ISR is an enhancement of SSG that allows for partial static regeneration of pages on demand.
    • With ISR, pages are still pre-built at build time, but specific pages can be regenerated in the background or on-demand when new data is available.
    • Pros: Combine benefits of SSG with the ability to update specific pages dynamically, improved user experience.
    • Cons: Complexity in managing data revalidation and regeneration, potential for stale data if not configured properly.

    React tips and tricks

    • Component names should be in uppercase.
    • Attributes in JSX are specified using a camelCase naming convention. For example, class becomes className and onclick becomes onClick.
    • Assign a unique key prop to each element in the list to help React identify which items have changed, are added, or are removed.
    • HTML element should have one parent, use either a div tag or fragment <></>.
    • All elements must be closed.
    • Use the === when comparing values.
    • When using CRA, project names need to be in lowercase.
    • Either .JSX or .JS file extensions can be used for Functional Components.
    • One Functional Component per file.
