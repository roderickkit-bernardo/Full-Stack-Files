Server Side JS

Node JS

  • Open-source, cross-platform JavaScript runtime environment that executes JavaScript code outside of a web browser.
  • It allows developers to use JavaScript to write server-side applications, enabling them to build scalable and high-performance network applications.
  • Built on Chrome's V8 JavaScript engine and uses an event-driven, non-blocking I/O model, making it efficient and lightweight for handling concurrent connections and I/O-intensive operations.

Node JS is not the only JS runtime, deno and bun are alternative JS runtimes.

Node JS Features

  • Asynchronous and Event-Driven:

    • Node.js operates on a non-blocking, event-driven architecture.
    • This means that it can handle a large number of concurrent connections efficiently, making it ideal for building scalable network applications.

  • Single-threaded but Highly Scalable:

    • Although Node.js runs on a single-threaded event loop, it can handle thousands of concurrent connections without the need for spawning new threads for each connection.
    • This makes it highly efficient in terms of resource utilization.

  • NPM (Node Package Manager):

    • Node.js comes with npm, a package manager that hosts over a million packages of reusable code.
    • This vast ecosystem of libraries and modules simplifies the process of building applications by providing ready-made solutions for common tasks.

  • Cross-Platform:

    • Node.js is cross-platform and can run on various operating systems such as Windows, macOS, and Linux, making it versatile for developers working on different environments.

  • Fast Execution:

    • Node.js is known for its fast execution speed due to the V8 engine, which compiles JavaScript into machine code before executing it.

  • Streaming Data:

    • Node.js provides built-in support for streaming data, allowing developers to process files and network streams efficiently in chunks, which reduces memory consumption and improves performance.

  • Server-side Web Development:

    • Node.js is commonly used for server-side web development. With frameworks like Express.js, developers can quickly build robust and scalable web applications.

  • Microservices Architecture:

    • Node.js is well-suited for a microservices architecture, where applications are broken down into smaller, independent services that communicate with each other through APIs.
    • Its lightweight nature and support for asynchronous operations make it an excellent choice for microservices.

    Application Architecture Types:

    Monolith:

    Pros:

    In a monolithic architecture, the entire application is built as a single, self-contained unit. All components of the application, including the user interface, business logic, and data access layer, are tightly integrated. Monolithic architectures are relatively simple to develop and deploy initially, as there is only one codebase to manage.

    Cons:

    Monolithic applications can become difficult to maintain and scale as they grow larger, because any change to one part of the application may require redeploying the entire monolith.

    Microservices:

    Pros:

    Microservices architecture decomposes an application into smaller, loosely coupled services that are independently deployable and scalable. Each microservice is responsible for a specific business function and communicates with other services through APIs. This approach offers flexibility, scalability, and resilience, as individual services can be developed, deployed, and scaled independently.

    Cons:

    Microservices introduce complexity in terms of managing communication between services, ensuring consistency across services, and monitoring and debugging distributed systems.

    Serverless:

    Pros:

    Serverless, also known as Function as a Service (FaaS), abstracts away the infrastructure management from developers. Developers focus on writing small, single-purpose functions that are triggered by events (e.g., HTTP requests, database changes). With serverless, developers don't need to provision or manage servers; the cloud provider automatically scales the infrastructure based on demand. Serverless architectures can be cost-effective, as users are only charged for the compute resources consumed by their functions.

    Cons:

    Serverless may not be suitable for all types of applications, and there may be limitations in terms of execution duration, resource constraints, and cold start latency.

  • Real-time Applications:

    • Node.js is often used for building real-time applications such as chat applications, online gaming platforms, and collaborative tools due to its ability to handle simultaneous connections efficiently.

  • Community and Support:

    • Node.js has a vibrant and active community of developers, which contributes to its ongoing development, provides support through forums and communities, and creates a wealth of resources for learning and troubleshooting.

Node Package Manager (npm)

  • Package manager for Node.js that comes bundled with the Node.js runtime environment.
  • It serves as a central repository for downloading and managing JavaScript libraries and tools.
  • npm allows developers to easily install, share, and manage dependencies for their Node.js projects.

Node Package Manager (npm) Features

  • Package Installation:

    • npm provides a command-line interface (CLI) that allows developers to install packages from the npm registry.
    • Developers can install packages globally (accessible from any project) or locally (specific to a project).

  • Dependency Management:

    • npm simplifies the process of managing dependencies for Node.js projects.
    • Developers can specify dependencies and their versions in a package.json file, and npm automatically installs the required packages and their dependencies.

  • Version Management:

    • npm allows developers to specify version ranges for dependencies in the package.json file.
    • This enables developers to specify a minimum and maximum version, ensuring compatibility with other packages while allowing for updates.

  • Publishing Packages:

    • Developers can publish their own packages to the npm registry, making them available for others to use.
    • This allows developers to share reusable code and contribute to the Node.js ecosystem.

  • Scoping:

    • npm supports scoped packages, which allow developers to organize packages under a specific namespace.
    • Scoped packages help prevent naming conflicts and make it easier to manage related packages.

  • Scripts:

    • npm allows developers to define custom scripts in the package.json file.
    • These scripts can be executed using the npm CLI, enabling tasks such as running tests, building the project, or deploying the application.

  • Security:

    • npm includes built-in security features to help protect against malicious packages.
    • It performs vulnerability scans on packages during installation and provides warnings for known security issues.

  • Versioning and SemVer:

    • npm follows Semantic Versioning (SemVer), which specifies version numbers in a structured format (major.minor.patch).
    • This helps developers understand the impact of updates and ensures compatibility between versions.

  • Registry Mirroring:

    • Organizations can set up private npm registries or mirror the public npm registry to host packages internally.
    • This allows teams to manage proprietary code and dependencies within their network.

Basic NPM commands:

  • npm init - Initialize a Node JS app.
  • npm init -y - Initialize a Node JS app and accept the defaults.
  • npm i NAME_OF_PACKAGE or npm install NAME_OF_PACKAGE - Installs NAME_OF_PACKAGE package. ie: npm i express or npm install express, will install the express package.
  • npm i -D NAME_OF_PACKAGE or npm install -D NAME_OF_PACKAGE - saves the package as a development dependency in the devDependencies section of your package.json file. ie: npm i -D nodemon or npm install -D nodemon, will install the nodemon package as a development dependency.
  • npm i or npm install - Install the packages listed under package.json dependencies list.

Express JS

  • Is a minimal and flexible web application framework for Node.js.
  • It provides a robust set of features for building web and mobile applications, as well as APIs (Application Programming Interfaces).
  • One of the most popular frameworks for Node.js due to its simplicity, flexibility, and performance.

Express JS Features

  • Middleware:

    • Middleware functions are at the core of Express.js.
    • They are functions that have access to the request object (req), the response object (res), and the next middleware function in the application's request-response cycle.
    • Middleware functions can perform tasks such as logging, authentication, data parsing, and error handling.

    Useful Middleware Functions:

    • app.use(express.static('public')); - Built in middleware that serves static files such as HTML, CSS, JavaScript, images, and other assets.
    • app.use(express.json()); - Built in middleware that parses the incoming request body if it contains JSON data and populates the req.body property with the parsed JSON object. This makes it easier to work with JSON data in your Express routes.
    • app.use(express.urlencoded({ extended: true })); - Built in middleware that parses the incoming request body if it contains URL-encoded data (e.g., data submitted from an HTML form) and populates the req.body property with the parsed data.
    • Third party middleware - Middleware functions or modules that are developed by the community or third-party developers to extend the functionality of Express applications. These third-party middleware modules can be installed via npm and then integrated into your Express application using app.use() just like built-in middleware.

  • Routing:

    • Express provides a simple yet powerful routing mechanism that allows developers to define routes based on HTTP methods (GET, POST, PUT, DELETE, etc.) and URL patterns.
    • Routes can handle requests and return responses, making it easy to create RESTful APIs and web applications.

  • HTTP Utilities:

    • Express provides utility methods and properties to work with HTTP requests and responses more efficiently.
    • These include methods for setting HTTP headers, sending status codes, redirecting requests, and sending various types of responses (JSON, HTML, etc.).

  • Static Files Serving:

    • Express makes it easy to serve static files such as HTML, CSS, JavaScript, images, and other assets.
    • Developers can use the express.static middleware to serve files from a specified directory, making it suitable for serving client-side resources in web applications.

  • Templating Engines:

    • Although Express itself does not include a built-in templating engine, it allows developers to integrate popular templating engines such as EJS (Embedded JavaScript), Handlebars, Pug (formerly known as Jade), and Mustache.
    • Templating engines enable the dynamic generation of HTML content based on data.

  • Error Handling:

    • Express provides built-in error handling mechanisms to catch and handle errors that occur during the request-response cycle.
    • Developers can define error-handling middleware functions to centralize error handling logic and improve code maintainability.

  • Modularity and Extensibility:

    • Express is designed to be modular and extensible, allowing developers to add additional functionality through third-party middleware and plugins.
    • This modularity makes it easy to integrate with other Node.js modules and frameworks, enhancing the capabilities of Express-based applications.

  • Community and Ecosystem:

    • Express has a large and active community of developers who contribute to its ongoing development, create middleware packages, and provide support through forums, documentation, and tutorials.
    • This vibrant ecosystem makes it easier for developers to build and maintain Express.js applications.

HTTP Method

  • HTTP (Hypertext Transfer Protocol) methods are verbs that define the actions that can be performed on a given resource.
  • They are an essential part of the HTTP protocol, used for communication between clients (such as web browsers) and servers.

HTTP Method List

  • GET:

    • The GET method requests data from a specified resource.
    • It should only retrieve data and not cause any other side effects.
    • It is a safe and idempotent method, meaning that multiple identical requests should have the same effect as a single request.

  • POST:

    • The POST method submits data to be processed to a specified resource.
    • It is often used to create new resources or to send data to a server for processing, such as submitting a form or uploading a file.
    • Unlike GET, POST requests may have side effects on the server and are not idempotent.

  • PUT:

    • The PUT method is used to update a resource or create a new resource if it does not exist at the specified URL.
    • It is idempotent, meaning that multiple identical requests should have the same effect as a single request.

  • DELETE:

    • The DELETE method is used to request the removal of a specified resource.
    • It is idempotent, meaning that multiple identical requests should have the same effect as a single request.

CSS Frameworks

  • CSS frameworks are pre-prepared libraries that contain standardized, pre-written code to help developers quickly and efficiently build web applications or websites.
  • They provide a set of CSS files that include a grid system, typography styles, form layouts, button styles, and other UI components.

CSS Frameworks Features

  • Grid System:

    • Most CSS frameworks come with a grid system that simplifies layout creation.
    • This grid system allows developers to create responsive layouts easily, ensuring that web pages look good on various devices and screen sizes.

  • Typography:

    • CSS frameworks often include predefined typography styles for headings, paragraphs, lists, and other text elements.
    • These styles ensure consistency and readability across the website.

  • UI Components:

    • Frameworks typically offer ready-to-use UI components such as buttons, forms, navigation bars, modals, and more.
    • These components are styled consistently and can be easily customized to match the design of the website.

  • Cross-browser Compatibility:

    • CSS frameworks handle cross-browser compatibility issues, ensuring that styles render consistently across different web browsers.

  • Customization:

    • While CSS frameworks provide pre-styled components, they also allow developers to customize styles according to their specific needs.
    • Most frameworks provide variables or configuration options to customize colors, spacing, typography, and other design aspects.

  • Responsive Design:

    • With the proliferation of mobile devices, responsive design has become essential.
    • CSS frameworks facilitate the creation of responsive layouts through their grid systems and responsive utility classes.

Popular CSS Frameworks

  • Bootstrap:

    • One of the most widely used CSS frameworks, Bootstrap provides a comprehensive set of UI components and a responsive grid system.

  • Bulma:

    • Bulma is a modern CSS framework based on Flexbox.
    • It emphasizes simplicity and modularity, making it easy to customize and use.

  • Tailwind CSS:

    • Unlike traditional CSS frameworks, Tailwind CSS provides utility classes that directly apply styles to HTML elements.
    • It offers a highly customizable and functional approach to styling web interfaces.

JS Templating Language

  • JavaScript templating languages are used to generate HTML markup dynamically on the client-side or server-side using JavaScript.
  • These languages provide a way to structure and organize HTML templates with placeholders that can be filled in with data during runtime.

Popular JS Templating Languages

  • Handlebars.js:

    • Handlebars.js is a popular templating engine that allows you to build semantic templates using a simple syntax.
    • It uses double curly braces {{ }} for placeholders and supports helpers and partials for more complex templating needs.
    • Handlebars.js is often used in conjunction with client-side frameworks like Ember.js or server-side frameworks like Express.js.

  • Mustache.js:

    • Mustache.js is a logic-less templating engine that follows the mustache syntax for placeholders, which consists of double curly braces {{ }}.
    • It is lightweight and easy to use, making it suitable for various JavaScript applications.
    • Mustache.js supports partials, lambdas, and sections for conditional rendering.

  • EJS (Embedded JavaScript):

    • EJS is a simple templating language that allows you to embed JavaScript code directly within HTML markup.
    • It uses delimiter tags <% %> to execute JavaScript code and <%= %> to output values.
    • EJS is often used with Node.js frameworks like Express.js for server-side rendering.

  • Pug:

    • Pug is a high-performance templating engine for Node.js and browsers that uses indentation-based syntax instead of HTML markup.
    • It allows you to write concise and readable templates with fewer characters.
    • Pug supports features like mixins, includes, and filters for creating modular and reusable templates.

The JS templating engine used for this class will be EJS (Embedded JavaScript). Feel free to use any templating engine of your preference.

Useful EJS Tags:

  • <% - 'Scriptlet' tag, for control-flow, no output
  • <%= - Outputs the value into the template (HTML escaped)
  • <%- - Outputs the unescaped value into the template

Includes are relative to the template with the include call. (This requires the 'filename' option.) For example if you have "./views/users.ejs" and "./views/user/show.ejs" you would use <%- include('user/show'); %>.

JS Modules

ES6 modules and CommonJS are both mechanisms for modularizing JavaScript code, but they have some differences in syntax, behavior, and usage.

  • Syntax:

    • ES6 Modules: Use import and export keywords to define dependencies and exports.

      Importing a module

      import { function1, function2 } from './module.js';

      Exporting from a module

      export { function1, function2 };

    • CommonJS: Uses require() to import modules and module.exports or exports to export values.

      Importing a module

      const module = require('./module');

      Exporting from a module

      module.exports = { function1, function2 };

  • Dynamic vs Static:

    • ES6 Modules: Are statically analyzable, meaning imports and exports are resolved at compile time. This allows for better optimization and tree-shaking (removing unused code) by build tools.
    • CommonJS: Are loaded dynamically at runtime, which can make it harder for build tools to perform optimizations like tree-shaking.

  • Browser vs Node.js:

    • ES6 Modules: Natively supported in modern browsers using import and export syntax. In Node.js, ES6 modules are supported since version 12 but must be used with the .mjs file extension or by configuring package.json.
    • CommonJS: Are the standard in Node.js and are also widely used in server-side JavaScript applications. In browsers, they require tools like Browserify or Webpack to bundle them for use.

  • Top-level Scope:

    • ES6 Modules: Each ES6 module has its own top-level scope, meaning variables and functions declared inside the module are not visible outside unless explicitly exported.
    • CommonJS: Modules share the same scope, so variables and functions declared in one module can be accessed by other modules unless explicitly scoped.
