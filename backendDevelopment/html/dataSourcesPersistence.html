Data Sources / Persistence

    SQL Databases

    • SQL databases, short for Structured Query Language databases, are a type of relational database management system (RDBMS) that use SQL as their query language.

    Key aspects of SQL databases include:

    Tables:

    • Tables are the core structures in a SQL database and consist of rows and columns where data is stored.
    • Each table represents a specific type of data, such as customers, orders, or products.

    Schema:

    • A schema defines the structure of the database, including tables, columns, data types, and relationships between tables.

    Primary Key:

    • A unique identifier for each record in a table. It ensures that each row can be uniquely identified.

    Foreign Key:

    • A column that creates a link between two tables. It is used to maintain referential integrity between the tables.

    SQL Queries:

    • SELECT: Used to retrieve data from one or more tables.
    • INSERT: Used to add new records to a table.
    • UPDATE: Used to modify existing records.
    • DELETE: Used to remove records from a table.
    • JOIN: Combines rows from two or more tables based on a related column between them.

    Examples of SQL Databases:

    MySQL:

    • Open-source database widely used in web applications.
    • Known for its reliability and ease of use.
    • Commonly used with PHP in the LAMP stack (Linux, Apache, MySQL, PHP).

    PostgreSQL:

    • Advanced open-source database known for its robustness, extensibility, and standards compliance.
    • Supports complex queries, foreign keys, triggers, and user-defined types.

    SQLite:

    • A lightweight, embedded SQL database engine.
    • Ideal for mobile apps, small to medium websites, and applications where simplicity and minimal setup are important.

    PostgreSQL:

    • Advanced open-source database known for its robustness, extensibility, and standards compliance.
    • Supports complex queries, foreign keys, triggers, and user-defined types.

    Microsoft SQL Server:

    • A comprehensive, enterprise-grade database management system by Microsoft.
    • Offers advanced features for security, performance, and scalability.
    • Integrated with other Microsoft products and services.

    Oracle Database:

    • A powerful and highly reliable enterprise-grade RDBMS.
    • Known for its performance, scalability, and advanced features such as Real Application Clusters (RAC) and Active Data Guard.
    • Integrated with other Microsoft products and services.

    MySQL is the Sql database used for this course.

    SQL Object Relational Mapper

    • An SQL Object Relational Mapper (ORM) is a tool that allows developers to interact with a database using an object-oriented paradigm instead of writing raw SQL queries.
    • ORMs provide a high-level abstraction that maps database tables to classes, and rows to objects of those classes.
    • This makes database operations more intuitive for developers who are familiar with object-oriented programming.

    SQL ORMs for different programming languages:

    Python:

    • SQLAlchemy: A powerful and flexible ORM that provides a full suite of enterprise-grade persistence patterns.
    • Django ORM: The built-in ORM of the Django web framework, known for its simplicity and ease of use.

    JavaScript (Node.js):

    • Sequelize: A promise-based ORM for Node.js that supports multiple SQL dialects and provides a robust feature set.
    • TypeORM: An ORM that supports both Active Record and Data Mapper patterns, written in TypeScript.

    Java:

    • Hibernate: A mature and widely used ORM that supports JPA (Java Persistence API) and provides advanced caching and performance features.
    • MyBatis: Unlike traditional ORMs, MyBatis couples objects with stored procedures or SQL statements using XML or annotations.

    Sequelize is the ORM used for this course.

    Benefits of Using an ORM:

    • Productivity: Developers can write less code compared to writing raw SQL, leading to faster development.
    • Maintainability: The code is easier to maintain since it is cleaner and more readable.
    • Portability: ORMs abstract away database-specific details, making it easier to switch between different database systems.
    • Security: ORMs can help prevent SQL injection attacks by using parameterized queries.

    No SQL Datatases

    • Are designed to provide flexible, scalable, and high-performance data storage solutions that differ from the traditional relational (SQL) databases.
    • They are particularly well-suited for handling unstructured or semi-structured data, large volumes of data, and applications requiring high throughput and low latency.

    Advantages of NoSQL Databases:

    • Scalability: Designed to scale out by distributing data across multiple servers.
    • Flexibility: Schema-less design allows for dynamic and evolving data models.
    • Performance: Optimized for high read and write throughput with low latency.
    • Big Data: Capable of handling large volumes of unstructured and semi-structured data.

    MongoDB is the NoSql database used for this course.

    No SQL Object Data Modelling

    • Is similar to Object Relational Mapping (ORM) used in SQL databases, but it's tailored for NoSQL databases.
    • ODMs map objects in your application to documents in a NoSQL database, allowing for more intuitive and structured interaction with the database.

    Key Features of ODMs for NoSQL Databases:

    • Schema Definition: Define the structure of documents and collections using a schema.
    • Data Validation: Ensure data integrity by validating data against the schema before saving.
    • CRUD Operations: Simplified methods for creating, reading, updating, and deleting documents.
    • Query Building: APIs to construct complex queries programmatically.
    • Middleware / Plugins: Support for middleware and plugins to extend functionality.
    • Relationships: Define and manage relationships between documents (e.g., references, embedded documents).

    Benefits of Using ODMs:

    • Ease of Use: Simplifies database interactions by using familiar object-oriented paradigms.
    • Maintainability: Promotes cleaner and more maintainable code.
    • Validation: Ensures data integrity through schema validation.
    • Consistency: Provides a consistent way to interact with the database, making it easier to manage changes.

    Mongoose is the ODM used for this course.

    Queues

    • Are fundamental data structures used extensively in computer science and software engineering to manage a collection of elements in a particular order.
    • They follow the First-In-First-Out (FIFO) principle, where the first element added to the queue is the first one to be removed.
    • Queues are essential for various scenarios, such as task scheduling, managing requests in web servers, handling asynchronous events.

    Applications of Queues:

    • Task Scheduling: Managing tasks in operating systems, job scheduling in servers.
    • Message Queuing: Handling communication between distributed systems (e.g., RabbitMQ, Kafka).
    • Breadth-First Search (BFS): Traversing or searching tree/graph data structures.
    • Caching: Implementing Least Recently Used (LRU) caches.
    • Load Balancing: Distributing tasks across multiple servers.

    Popular Queue Libraries and Services:

    • RabbitMQ: An open-source message broker that implements the Advanced Message Queuing Protocol (AMQP).
    • Apache Kafka: A distributed event streaming platform capable of handling high-throughput and low-latency messaging.
    • Amazon SQS: A fully managed message queuing service that enables decoupling and scaling of microservices, distributed systems, and serverless applications.
    • Redis: An in-memory data structure store that supports list operations, which can be used to implement queues.

    Redis and Rabbitmq are the queues used for this course.
